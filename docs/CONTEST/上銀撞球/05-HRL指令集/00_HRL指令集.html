🏃運動指令

PTP — 點對點移動

格式說明
PTP {X 100, Y 0, Z 200} CONT=100% Vel=100% Acc=50% TOOL[0] BASE[0] 笛卡爾座標絕對移動。A1~A6 或 X,Y,Z,A,B,C 皆可。
PTP {A1 45} CONT=100% Vel=100% Acc=50% 關節角度絕對移動（單軸或多軸）。
PTP $PR[1] CONT=100% Vel=100% 使用位置暫存器移動。
⚠️ 注意：Vel 對於 PTP 是百分比（0~100%），對於 LIN/CIRC 是 mm/s。

PTP_REL — 點對點相對移動

格式說明
PTP_REL {X 100} CONT=100% Vel=100% Acc=50% 相對於目前位置的增量移動（笛卡爾或關節）。

LIN — 直線移動

格式說明
LIN {X 100, Y 0, Z 200} CONT=100% Vel=2000mm/s Acc=50% TOOL[0] BASE[0] TCP 沿直線移動到目標點。Vel 單位為 mm/s。
LIN $PR[1] CONT=100% Vel=2000mm/s 使用位置暫存器直線移動。

LIN_REL — 直線相對移動

格式說明
LIN_REL {X 100} CONT=100% Vel=2000mm/s Acc=50% 直線增量移動（笛卡爾）。

LIN_REL_TOOL — 工具座標直線相對移動

格式說明
LIN_REL_TOOL {X 100} CONT=100% Vel=2000mm/s Acc=50% 沿工具座標系方向做直線相對移動。一次只能對一個軸操作。

CIRC / CIRC_REL — 圓弧移動

格式說明
CIRC POINT1 POINT2 CONT=100% Vel=2000mm/s Acc=50% POINT1=輔助點（圓弧經過點），POINT2=目標點。兩者同為 POS 或同為 DEG。
CIRC {X 0, Y 450} {X -150, Y 300} CONT=100% 直接用笛卡爾座標指定圓弧。

SPLINE — B-Spline 樣條運動

格式說明
SPLINE
SPL P1
SPL P2
SPL P3
ENDSPLINE		 多點擬合 B-Spline 曲線運動。P1~P73 可用 E6POINT 型態。

運動參數說明

參數說明預設值
CONT過渡模式：0=FINE（精確到點），1~99=平滑過渡百分比FINE
Vel速度：PTP 為百分比(%)，LIN/CIRC 為 mm/s100% / 200mm/s
Acc加速度百分比（0~100%）50%
TOOL[n]工具座標系編號（0=無法規）TOOL[0]
BASE[n]基座座標系編號（0=世界座標）BASE[0]
FINE vs CONT：FINE 會完全停止在該點再執行下一行指令。CONT（1~99）則在抵達前就開始過渡到下一個運動，可能節省時間但軌跡不平滑。

📡通訊指令（TCP/IP · RS232）

Ethernet TCP/IP

指令說明
COPEN(ETH, HANDLE) 以 Client 模式連線（單一連線）。需先在 HRSS 設定好 IP/Port。
HANDLE = COPEN(ETH, 192, 168, 0, 5, 5000) 以 Client 模式直接指定 IP (192.168.0.5) 和 Port (5000)。回傳 HANDLE。
CREAD(HANDLE, val1, val2, ...) 讀取網路資料（最多 400 筆），封包格式 {val1,val2,...}。讀取數量不足時，未對應的變數為 0。
CWRITE(HANDLE, val1, val2) 傳送資料（最多 400 筆）。每個 CWRITE 都是一組封包。val 只能是 REAL 或 STRING。
CCLEAR(HANDLE) 清除接收緩衝區。封包數量 > CREAD 指令數量時，需 CCLEAR 避免讀到舊資料。
CINQUIRE(HANDLE, NUM) 查詢緩衝區中目前的封包數量。
CCLOSE(HANDLE) 關閉通訊連線。

RS232 序列

指令說明
COPEN(SER, HANDLE)開啟 RS232 通訊。
CREAD / CWRITE / CCLEAR / CINQUIRE / CCLOSE用法與 Ethernet 版本相同。

典型 Python ↔ KRL 通訊流程

;; KRL 端 — TCP/IP Client 接收座標
INT HANDLE
REAL X, Y, Z, A, B, C
HANDLE = COPEN(ETH, 192, 168, 1, 100, 5000)  ; 手臂為 Client，連向 Python

LOOP
  CINQUIRE(HANDLE, NUM)
  IF NUM > 0 THEN
    CREAD(HANDLE, X, Y, Z, A, B, C)
    $PR[1] = {X X, Y Y, Z Z, A A, B B, C C}
    LIN $PR[1] Vel=500mm/s Acc=50%
    CWRITE(HANDLE, 1)   ; 回傳「到位完成」
  ENDIF
  CCLEAR(HANDLE)
  WAIT SEC 0.1
ENDLOOP
封包格式：預設用 {} 包覆，, 分隔。例如 {100,200,45} 會被解讀為 3 個數值。可在 HRSS 設定中改為 Non-Format（純 ASCII）。

📦暫存器指令

暫存器型態說明範例
$PR[#] E6POS / E6AXIS / E6POINT 位置暫存器。儲存 X,Y,Z,A,B,C（笛卡爾）或 A1~A6（關節）。 $PR[1] = {X 100, Y 0, Z 200}
$C[#] INT Counter 計數暫存器。可自由讀寫，用於計數、旗標。 $C[1] = $C[1] + 1
$DI[#] BOOL Digital Input。讀取數位輸入訊號（外部感測器、極限開關等）。 IF $DI[1] == TRUE THEN ...
$DO[#] BOOL Digital Output。控制數位輸出訊號（電磁閥、燈號等）。 $DO[1] = TRUE
$RI[#] BOOL Robot Input。機器手臂專用輸入。 WAIT FOR $RI[1] == TRUE
$RO[#] BOOL Robot Output。機器手臂專用輸出。 $RO[1] = TRUE
$T[#] REAL Timer 計時暫存器（秒）。需搭配 $T_STOP 啟動。 $T[1] = 0 · $T_STOP[1] = TRUE
$VO[#] BOOL Valve 電磁閥輸出。 $VO[1] = TRUE

$PR 賦值方式

;; 關節角度賦值
$PR[1] = {A1 1, A2 2, A3 3, A4 4, A5 5, A6 6}

;; 笛卡爾座標賦值
$PR[2] = {X 7, Y 8, Z 9, A 10, B 11, C 12}

;; 用 E6POS / E6AXIS / E6POINT 間接賦值
E6POS A = {X 10, Y 10, Z 10, A 10, B 10, C 10}
$PR[1] = A

;; 用 GETPOINT 取得目前位置
$PR[1] = GETPOINT

特殊：PAL_PR 的 $C 用法

手冊中 PAL_PR 函式使用 $C[INDEX+15] 儲存格點數量、$C[16]~$C[30] 為各 block 的 offset。這是 HIWIN Library 自訂函式（非標準 HRL）的內部用法，與一般計數用途分開。

🔤變數型態指令

型態說明範例
BOOL布林（TRUE / FALSE）BOOL K = TRUE
INT帶正負號整數INT I = 0
UINT無符號整數UINT N = 100
REAL浮點數REAL R = 3.14
CHAR單一字元CHAR C = 'A'
STRING字串STRING S = "HIWIN"
E6POS 直角座標系位置（X, Y, Z, A, B, C） E6POS P = {X 0, Y 300, Z 200}
E6AXIS 關節座標系角度（A1~A6, E1~E3） E6AXIS A = {A1 0, A2 0, A3 0}
E6POINT 可存座標或角度（混用時缺失欄位為 0） E6POINT HOME = {Y 200, Z -1000, A 90}
FRAME 座標框架（含原點+姿態） FRAME F = {X 100, Y 200, A 90}

E6POS / E6AXIS 差異

;; E6POS = 笛卡爾座標（姿態）
E6POS P1 = {X 0, Y 300, Z 200, A 0, B 0, C 0}

;; E6AXIS = 關節角度
E6AXIS J1 = {A1 0, A2 -30, A3 50, A4 0, A5 90, A6 0}

;; E6POINT = 兩者皆可，缺失欄位自動填 0
E6POINT P2 = {Y 200, Z -1000, A 90}   ; X=0, B=0, C=0

;; 兩者可相互賦值
$PR[1] = P1   ; E6POS → $PR
$PR[2] = J1   ; E6AXIS → $PR

🔀控制流程指令

指令說明範例
IF / THEN / ENDIF單一條件判斷IF n > 0 THEN PTP P1 ENDIF
FOR / ENDFOR計次迴圈FOR i = 1 TO 10 STEP 2 ... ENDFOR
LOOP / ENDLOOP無限迴圈（需搭配 EXIT 離開）LOOP ... IF X THEN EXIT ENDIF ... ENDLOOP
WHILE / ENDWHILE前測試迴圈（條件成立才執行）WHILE $DI[1] == TRUE ... ENDWHILE
REPEAT / UNTIL後測試迴圈（至少執行一次）REPEAT ... UNTIL n > 2
SWITCH / CASE / ENDSWITCH多條件分支SWITCH n CASE 0: ... CASE 1: ... ENDSWITCH
GOTO跳至標記位置GOTO FOUND（需先有 FOUND: 標記）
WAIT FOR等待條件成立WAIT FOR $DI[1] == TRUE
WAIT SEC等待秒數WAIT SEC 0.5
QUIT結束程式執行IF error THEN QUIT ENDIF
DEF / END定義副程式DEF MOVE_LOOP() ... END
DEFFCT / ENDFCT定義函式（可回傳值）DEFFCT INT MY() ... RETURN 100 ... ENDFCT

典型運動+等待迴圈

DEF MOVE_LOOP()
  WHILE TRUE
    CREAD(HANDLE, X, Y, Z, A, B, C)
    $PR[1] = {X X, Y Y, Z Z, A A, B B, C C}
    LIN $PR[1] Vel=500mm/s Acc=50%
    $DO[1] = TRUE    ; 通知 Python 已到位
    WAIT SEC 0.3
    $DO[1] = FALSE
  ENDWHILE
END

📚HIWIN Library 函式（.hrb 預編譯模組）

⚠️ 重要：以下函式屬於 HIWIN Library 自訂模組（.hrb 檔），非標準 HRL 指令集一部分。需將 .hrb 檔案匯入手臂系統後才能使用。詳細用法請見 HIWIN_Library.zip 說明文件。
函式說明筆記
JUMP(PPE, H) 安全移動：先上升到 H 高度，再水平移動到 PPE，最後下壓。 適合拿來做「球旁安全接近」
ATPOS(PPE, N) 確認手臂是否移動到目標位置（誤差範圍內）。到位後設定 $DO[N]=TRUE。 可用於 Python 讀取 $DO 確認到位
BOX / INSIDEBOX BOX 設定擊球範圍邊界；INSIDEBOX 檢查 PR 位置是否在範圍內。 可用於確認球是否在手臂可及範圍
PAL_PR(INDEX, ROW, COL, PP1, PP2, PP3) 雙線性內插產生格點，存入 $PR[3501+N]。使用 $C[INDEX+15] 存格點數。 用於預先產生球桌格點資料庫
CIRCLE / OVAL 圓弧/橢圓軌跡（與 SPLINE 不同，這是封閉曲線） 撞球路徑規劃可能用不到
ABS / SQRT / MOD / ROUND 數學運算（與標準 HRL 指令重疊） 與標準指令功能類似
BIT(DI, n) 讀取第 n 個數位輸入的狀態 相當於 $DI[n]
GETPOINT 存入目前 TCP 位置（與標準 HRL GETPOINT 相同） 標準 HRL 已有，Library 版無差異

PAL_PR 詳細邏輯（給有興趣的人）

; 輸入：INDEX=block編號, ROW×COL=格點大小, PP1/PP2/PP3=三個角點
; 輸出：$PR[3501 + offset + K] = 格點位置（K=1~N）
;
; 三個角點定義格點範圍：
;   PP2 ─── PP3
;    │ I 軸 │
;    │ ↓   │
;   PP1
;
; 公式（雙線性內插）：
;   PAL.X = PP1.X + (PP2.X-PP1.X)/(ROW-1)*I + (PP3.X-PP1.X)/(COL-1)*J
;   PAL.Y = PP1.Y + (PP2.Y-PP1.Y)/(ROW-1)*I + (PP3.Y-PP1.Y)/(COL-1)*J
;   PAL.Z = PP1.Z + (PP2.Z-PP1.Z)/(ROW-1)*I + (PP3.Z-PP1.Z)/(COL-1)*J

與撞球相關的使用建議

  • JUMP：手臂從待機位置移動到球旁的安全路徑
  • ATPOS：Python 輪詢 $DO[1] 確認到位，再發送擊球指令
  • INSIDEBOX：擊球前確認球在手臂範圍內，避免白白浪費動作
  • PAL_PR：若要在手臂端預先建立球桌格點庫（教球），可用此函式
