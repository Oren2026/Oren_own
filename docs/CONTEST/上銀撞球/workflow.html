測試流程手冊

進實驗室前從頭跑一遍，確保各模組正常再進行整合測試

測試流程
P0
環境確認
Terminal 指令 每次進實驗室
▼
  • 1
    確認工作目錄
    所有指令都在這個目錄下執行
  • 2
    確認 Python 環境與依賴
    確認版本、安裝必要套件
  • 3
    確認網段（Mac mini ↔ HIWIN 手臂）
    兩者在同一網段才能 TCP 連線
  • 4
    確認 Arduino COM Port
    插入前後各查一次 /dev/cu.*，確認路徑
  • 5
    確認 webcam ID
    CAM_GLOBAL=0 為第一顆，1 為第二顆
Terminal 指令
cd ~/Desktop/Oren_own/hiwin_pool python3 --version pip install -r requirements.txt # 測試手臂網段 ping -c 1 192.168.1.100 # 找 Arduino COM Port ls /dev/cu.* # 測 webcam python3 -c "import cv2; cap=cv2.VideoCapture(0); print('CAM0 OK' if cap.isOpened() else 'FAIL'); cap.release()"
⚠️
COM Port 每次插拔可能會變。更換 USB 埠或重新插拔後，务必重新確認路徑，並更新 config/strike_config.py 的 ARDUINO_COM。
P1
Webcam 影像確認
視覺模組 每次調整鏡頭後
▼
  • 1
    執行 ball_test_tool.py
    直接觀察 webcam 影像品質
  • 2
    確認球檯完整出現在畫面中
    四個角落都必須在視角範圍內
  • 3
    觀察光源是否穩定
    無明顯反光或陰影飄動為佳
  • 4
    觀察白球是否為畫面中最大面積的白色物件
    如果不是，調整光源或視角
執行指令
python3 vision/ball_test_tool.py
影像流暢無明顯延遲
球檯四個角落都在畫面中
光源穩定無嚴重反光
白球為最大面積白色物件
⚠️
這個階段 不要求球偵測正確，只確認能看到、看得清楚。球偵測問題在 Phase 3 處理。
💡
控制鍵：[g] 開關 Homography 網格　[s] 截圖　[q] 結束
P2
Homography 校正
視覺模組 鏡頭移動後必做
▼
  • 1
    選擇校正模式
    Marker 模式（快速）：鏡頭沒動過，用球檯已知尺寸自動計算
    Manual 模式（通用）：鏡頭有移動，重新量測 mm 座標
  • 2
    依序點擊球檯四個角落
    順序：左上 → 右上 → 右下 → 左下（由上而下、由左而右繞一圈）
  • 3
    Marker 模式：自動計算，結束
    Manual 模式：依序輸入各點的世界座標（mm）
  • 4
    回到 Phase 1，按 [g] 開啟網格驗證
    格線應與球檯邊線重疊、直線無扭曲
執行指令
python3 vision/homography_calibrator.py
4個角都正確點到球檯邊緣
網格線與球檯邊線重疊
格線為直線（無明顯扭曲）
終端機最大重投影誤差 < 10mm
⚠️
每次移動 webcam 後都必須重新校正。球檯尺寸是固定的（1200×630mm），不需要每次都量，只要鏡頭位置固定，用 Marker 模式即可快速重新校正。
💡
球檯坐標系：x=0 在左側（近桿弟），y=0 在近端（球員側）。原點在球檯左下角。
P3
白球偵測參數調整
視覺模組 首次設定 / 光線變化後
▼
  • 1
    執行 ball_test_tool.py，觀察偵測結果
    白球應有黃色十字標記，桌面座標應在 1200×630mm 範圍內
  • 2
    觀察是否有誤偵測
    數字周圍白斑應被 BALL_MIN_AREA 過濾，不應出現大量雜點
  • 3
    調整 HSV 閾值或面積閾值
    問題嚴重時修改 camera_config.py，再重跑驗證
執行指令
python3 vision/ball_test_tool.py
調整參數 — config/camera_config.py
# HSV 閾值（白球：低彩度、高亮度）
BALL_COLOR_LOWER = {"h": 0, "s": 0, "v": 180} ← 提高 V 下限
BALL_COLOR_UPPER = {"h": 180, "s": 60, "v": 255}

# 面積閾值（像素²）
BALL_MIN_AREA = 200 ← 如數字白斑漏過，提高此值
💡
面積參考：白球直徑≈50mm，在 640×480 webcam 下≈30-50像素半徑，面積≈2500-8000像素²。數字白斑通常遠小於此值。
P4
手臂通訊確認
通訊模組 每次開機後
▼
  • 1
    確認手臂已開機並處於遠端控制模式
    手持終端無緊急停止燈號
  • 2
    測試 TCP 連線
    確認 IP 正確（192.168.1.100）、Port 正確（5000）
  • 3
    測試讀取手臂位置
    確認 get_position() 回傳合理 XYZ 座標、Z > 50mm
Python 指令（分開執行）
# 測試連線 python3 -c " from comm.hiwin_arm import HIWINArm arm = HIWINArm() ok = arm.connect() print('手臂連線:', '成功' if ok else '失敗') if ok: arm.disconnect() " # 讀取位置 python3 -c " from comm.hiwin_arm import HIWINArm arm = HIWINArm() arm.connect() pos = arm.get_position() print('位置:', pos) arm.disconnect() "
TCP 連線成功
get_position() 回傳合理數值
Z軸 > 50mm（在安全範圍）
無緊急停止燈號
🚨
緊急處理：如手臂失控，立即按下手持終端上的緊急開關。確認安全後再繼續。
P5
Arduino / 桿弟機構確認
末端裝置 每次開機後
▼
  • 1
    確認 Arduino COM Port 並更新設定
    插入前後各查一次 /dev/cu.*，找到 usbmodem 開頭的路徑
  • 2
    測試序列通訊
    確認 Python 能與 Arduino 正常來回
  • 3
    復位一致性測試（關鍵）
    連續復位 3 次，觀察每次是否穩定完成
  • 4
    確認馬達供電正常、無異常聲音
    供電不足會導致復位不一致
更新 COM Port — config/strike_config.py
ARDUINO_COM = "/dev/cu.usbmodem14101" # ← 替換為實際路徑
Python 指令（分開執行）
# 測試序列通訊 python3 -c " from comm.arduino_serial import ArduinoController ard = ArduinoController() ok = ard.connect() print('Arduino 連線:', '成功' if ok else '失敗') if ok: st = ard.read_status() print('狀態:', st) ard.disconnect() " # 復位一致性測試（關鍵！） python3 -c " from comm.arduino_serial import ArduinoController import time ard = ArduinoController() ard.connect() for i in range(3): r = ard.reset() print(f'第{i+1}次復位:', '成功' if r['complete'] else f'失敗: {r[\"error\"]}') time.sleep(1) ard.disconnect() "
3次復位皆成功完成
復位時間穩定（一致）
馬達無異常聲音
COM Port 已更新
⚠️
復位不一致是最常見的問題。可能原因：馬達供電不足、桿弟機構有機械干涉、Arduino 供電不穩。逐一排查。
P6
完整整合測試
整合驗證 所有 Phase 完成後
▼
  • 1
    確認所有前置 Phase 已完成
    P1 → P2 → P3 → P4 → P5 皆已驗證正常
  • 2
    執行 test_arm_to_ball.py
    空白鍵：偵測球 → 手臂移到白球上方（Z=200mm 固定）
  • 3
    觀察手臂是否移動到正確位置
    從 webcam 影像確認手臂在球正上方（Z不應下降）
  • 4
    確認終端機顯示 Result: OK
    表示手臂已成功到達目標位置
執行指令
python3 test_arm_to_ball.py
控制鍵
[空白鍵] 偵測球 → 移動手臂到白球上方
[r] 復位 Arduino（桿弟機構）
[q] 結束
💡
移動邏輯：手臂先 PTP 到 (Bx, By, 300mm) 上升到安全高度，再 LIN 到 (Bx, By, 200mm) 下降到懸停高度。Z軸先升後降是為了避免桿弟機構擦到球檯。
球偵測座標穩定
手臂在球正上方
Z軸維持 200mm（未下降）
終端機顯示 Result: OK
P7
UI 面板確認
UI 模組 首次設定後
▼
  • 1
    確認 12 段力道都能選擇
    力道 1-12（弱/中/強各有 4 段）
  • 2
    確認 Arduino Reset 按鈕正常
    按下後機構應完成復位
  • 3
    確認選單不會卡住
    UI 能正常響應所有操作
測試指令
python3 -c "from ui.tune_panel import TunePanel; print('UI 模組 OK')"
💡
UI 面板為輔助工具，正常測試流程中不一定要開，但進度展示或對外說明時會用到。
📋

參數速查

參數 預設值 所在檔案
HIWIN_IP 192.168.1.100 config/arm_config.py
HIWIN_PORT 5000 config/arm_config.py
ARDUINO_COM COM3（需確認） config/strike_config.py
SAFE_Z_HEIGHT 200mm config/arm_config.py
STRIKE_Z_HEIGHT 50mm config/arm_config.py
Z_MIN / Z_MAX 50 / 300mm config/arm_config.py
CAM_GLOBAL 0 config/camera_config.py
BALL_MIN_AREA 200 px² config/camera_config.py
TABLE_WIDTH / HEIGHT 1200 × 630mm config/camera_config.py
力道段數 1-12（共12段） config/strike_config.py
BAUD_RATE 115200 config/strike_config.py
PERSPECTIVE_MATRIX 校正後寫入 config/camera_config.py
⚡

最短流程（時間有限時）

P1 ball_test_tool.py → P2 homography_calibrator.py → P6 test_arm_to_ball.py
Phase 4、5 如已確認過、且機械結構未變動，可以跳過。
🔧

快速疑難排解

問題 檢查順序
Webcam 無影像 確認 CAM_GLOBAL 是否正確（0 或 1）
手臂連線失敗 確認 IP/Port 正確、手臂已開機、網段相同
Arduino 無法連線 確認 COM Port 正確（每次插拔都可能變）
球偵測不到 調整 HSV 閾值或 BALL_MIN_AREA
Homography 誤差大 確認4點點擊位置準確、光線穩定
手臂不動 確認處於遠端控制模式、非緊急停止狀態
復位不一致 檢查馬達供電、機械干涉、Arduino 供電
