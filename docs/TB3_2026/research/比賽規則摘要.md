# 比賽規則摘要

## 賽道總覽

賽道為封閉式迴圈設計，採逆時針方向行進。兩側有黃線（左側）與白線（右側）供車道跟隨使用。

## 任務流程（依行進順序）

1. **起點線（紅線）** — 機器人出發位置，方向朝左
2. **交通號誌** —  起點左側三色號誌燈，測試紅燈停、綠燈行
3. **圓環/交叉路口** — 藍色圓形箭頭標誌，決定出口方向
4. **減速帶/障礙區** — 左下角三條青藍色橫向標記
5. **停車區** — 底部中段岔路，白色虛線車格，藍色 P 標誌入口
6. **窄路與平交道柵欄** — 右下角紅白相間柵欄
7. **S 型彎道** — 右側連續彎道
8. **隧道/暗區** — 右上方大面積黑色區域
9. **返回終點線** — 繞完一圈回到起點

## 地面標線

| 顏色 | 位置 | 意義 |
|------|------|------|
| 黃線 | 行進方向左側 | 車道左邊界 |
| 白線 | 行進方向右側 | 車道右邊界 |
| 紅線 | 起點/終點 | 起終點線 |

## 核心技術

| 技術 | 用途 |
|------|------|
| HSV 顏色空間 | 紅/黃/綠燈號偵測，比 BGR 更穩定 |
| 霍夫圓變換 | 在顏色遮罩中找圓形，確認是否為真實燈號 |
| 透視轉換 | 將視角轉為 top-down，擠出消失點 |
| 滑動視窗 | 找出左側黃線、右側白線，計算中心線 |
| PID 控制 | 偏離中心線輸出 steering angle |
| 光照補償 | 隧道區曝光補償，須有備援機制 |

## 各關卡說明

詳細實作狀態請見 [系統架構](../architecture/系統架構規劃.html) 頁面。

| 關卡 | 說明 |
|------|------|
| 🚦 交通號誌 | 辨識紅/黃/綠燈，控制停與行 |
| ↩️ 圓環路口 | 識別方向箭頭，決策出口 |
| 🚧 障礙區 | 檢測障礙物，減速或繞行 |
| 🅿️ 停車區 | 定點停入車格 |
| 🚞 平交道 | 識別柵欄與火車，等待通行 |
| 〰️ S 型彎道 | 跟隨彎道中心線行進 |
| 🚇 隧道 | 暗區行駛，光照補償失效時備援 |
| 🚗 車道跟隨 | 基礎車道線偵測 + PID 控制 |
