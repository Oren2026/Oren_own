🚀 快速安裝

安裝依賴

進入 backend 資料夾，安裝 Node.js 相關套件

cd backend
npm install

設定環境變數

複製 .env.example 為 .env，並填入 JWT_SECRET（長度愈長愈好）

cp .env.example .env
# 用文字編輯器打開 .env，設定 JWT_SECRET

啟動伺服器

第一次啟動會自動建立空的 SQLite 資料庫

npm start

開啟瀏覽器

访问 http://localhost:3000

⚠️ 注意：本專案需要 Node.js 環境執行，無法直接以靜態檔案在瀏覽器開啟。別人若要使用，需具備基本的命令列操作能力（clone → npm install → npm start）。

❓ 常見問題

Q：啟動出現 `Error: listen EADDRINUSE`

Port 被佔用。改用其他 port：PORT=3001 npm start，或先 lsof -i :3000 找程序砍掉。

Q：第一次進去是一片空白

確認 npm install 有執行，且 node_modules/better-sqlite3 有正確編譯（需要 C++ 編譯器）。macOS 上可用 xcode-select --install 安裝編譯工具。

Q：想改成 admin 帳號

用 SQLite 工具直接改資料庫：UPDATE users SET role = 'admin' WHERE username = '你的帳號';

Q：想看資料庫內容

sqlite3 chatrank.db ".tables"
sqlite3 chatrank.db "SELECT * FROM users;"
💬 開發紀錄

Gravity 熱門排序

公式：score = likes / POWER(hours + 2, gravity)，gravity 範圍 0.5 ~ 3.0，透過 UI slider 可動態調整。

Rate Limit 修復（ngrok 共用 IP 問題）

原本 rate limit 用 IP 當 key，但 ngrok 多人共用同一出口 IP 會誤判。修復：改用 userId 當 rate limit key，未登入走 IP fallback。

Bug 修復（#1 ~ #15）

CSRF token 失效 → 改用 HMAC-SHA256 綁定 userId；likedSet key type 錯誤 → 統一 "userId:postId" 字串格式；Race condition → likedSet 檢查 + 資料庫事務隔離；comment-btn 無效 → 事件監聽器 attach 時機修正；renderPostDetail race condition → 加入 AbortController。
