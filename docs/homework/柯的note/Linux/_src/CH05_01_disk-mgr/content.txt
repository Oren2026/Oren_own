# CH05 磁碟與檔案系統管理 投影片原文

## 學習目標
使用 Linux 磁碟與檔案系統管理是日常會用到的技能，本章將介紹如何掛載 USB 裝置、光碟（或 ISO 檔）、安裝第 2 顆硬碟、檢查磁碟的使用狀況及連結與符號連結的使用。

## S1：掛載 USB 裝置

### USB 隨身碟掛載基礎
在 Linux 的文字模式下要使用 USB 裝置，使用者需要執行掛載的指令，才可讀寫資料。掛載，就是將儲存媒體（如光碟或硬碟）指定成系統中的某個目錄。

### 掛載指令
使用 **sudo mount** 指令以管理者權限執行。將 USB 隨身碟掛載在 /media/usbdisk 目錄，但預設並沒有此目錄，以 **sudo mkdir /media/usbdisk** 指令建立。

### 查詢與掛載 USB 裝置
- 查詢 USB 隨身碟的裝置名稱
- 執行 mount 指令，將隨身碟掛載至系統中
- 透過存取該目錄來讀取隨身碟上的資料

### 掛載外接式硬碟或讀卡機
掛載之前，必須先建立好對應的目錄。舉例：有一個 2TB 的外接式硬碟，其中又分成 2 個 1TB 的分割區，先建立 /media/usbdisk、/media/usbdisk2 這 2 個目錄。

### 卸載指令 (umount)
使用 umount 指令卸載。若無法成功卸載隨身碟，執行 **fuser -u /media/usbdisk/** 指令，可以查出使用該目錄的程序與使用者。

### 一般使用者掛載權限
Linux 預設只允許 root 使用者執行 mount 指令，一般使用者執行上述指令會出現錯誤訊息。開放一般使用者掛載隨身碟，以 root 使用者執行 sudo vi /etc/sudoers 指令修改設定。掛載使用 **sudo mount /dev/sdb1 /media/usbdisk**，卸載則是使用 **sudo umount /media/usbdisk**。

## S3：檢查磁碟使用狀況

### df 指令
介紹查詢磁碟剩餘空間的 df 指令與監控磁碟讀寫狀況的 dstat 指令。
- **df -h**：會將數值除以 1024 以方便閱讀
- **df -T**：顯示檔案系統的格式，例如：ext3、ext4...

### 檢查磁碟空間
檢查磁碟空間的使用狀況，列出占用空間的 10 大目錄。

### dstat 指令
dstat 指令也可以監看 CPU（使用 "-c" 參數）與網路（使用 "-n" 參數）的讀寫狀態。

### iotop 指令
使用 iotop 指令來查看是哪個使用者或程式在使用磁碟 I/O。若有懷疑的使用者，可用 "-u" 參數加上使用者名稱。

## S4：連結與符號連結

### ln 指令介紹
本節說明如何使用 ln (link) 指令建立檔案的硬連結與符號連結，以及使用的時機。執行 ls -l 指令列出檔案資訊時，會看到相關資訊。

### 符號連結與硬連結
在 Linux 的檔案系統中，比較常見的是符號連結 (symbolic link)。若有兩個（或以上）檔案使用相同的 inode，稱之為硬連結 (hard link)。

### 硬連結、符號連結與複製檔案的差別
說明三者的差異。

### 建立硬連結
單純的檔案複製方式會佔用磁碟空間。建立 **硬連結 (hard link)**：執行 ls -l 指令，比較複製的 LambertFile 與硬連結的 LambertLink 的不同。執行 **ls -i** 指令，觀察硬連結的檔案 inode 編號是否一樣。用複製而非連結的方法，便會是兩個不相干的檔案，各自擁有其 inode 編號。

### 建立符號連結
**ln -s** 指令建立符號連結。執行 ls -l 指令觀看 lambert 目錄的情形。符號連結本身也會佔用一個 inode。

### 硬連結與符號連結的差異
由於連結的方式不同，硬連結與符號連結有著以下的差異：
- 當原檔刪除後，符號連結將會失效，**硬連結則仍然可以繼續使用**
- **硬連結只能連結同一個分割區內的檔案**，而符號連結可以跨越不同分割區
- **硬連結不能連結目錄**，符號連結可以指向任何目錄

### 版本相關檔案的使用時機
與版本相關的檔案，一般會使用符號連結。

### Windows 上的連結指令
在 Windows 下可以使用 mklink 指令來建立硬連結與符號連結。

## S5：安裝第 2 顆硬碟

### 安裝新硬碟
Linux 操作一段時間後，隨著檔案的增加，您可能會發現原來的硬碟不夠用了，需要加裝一個新的硬碟才行。接下來介紹如何在 Linux 安裝第 2 顆硬碟。

### 硬碟分割
以 **fdisk** 指令來分割一台連接於 /dev/sdb 的 1TB 硬碟為例。

### 格式化與掛載
- **格式化硬碟**：執行 mkfs.ext4 指令
- **掛載成某個目錄**
- **開機時自動掛載**：修改 /etc/fstab 檔

## S6：Swap 分割區

### Swap 分割區概述
Ubuntu 在安裝時若選擇讓系統自動分割硬碟，系統並不會特別的建立一個獨立的 swap 分割區，取而代之的是在系統的根目錄下建立一個名為 swapfile 的 swap 檔案。

### 新增 Swap 分割區
利用 **fdisk 指令**新增 swap 分割區，將分割區的檔案系統更改為 swap。

### 格式化及啟動 Swap 分割區
執行 **mkswap** 指令格式化，執行 **swapon -s** 指令檢查。也可以執行 **free** 指令檢查 swap 是否增加。

### 開機時自動啟動
修改 /etc/fstab 檔即可在開機時自動掛載。

### 建立 Swap 檔案
要建立 swap 檔案，請執行 **dd 指令**。執行 **mkswap 指令**，將 myswap 檔案格式化成 swap 檔案系統，系統才能使用。修改 /etc/fstab 檔即可在開機時自動掛載。
