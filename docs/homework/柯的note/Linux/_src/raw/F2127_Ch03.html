• shell 的功能在於搭起使用者與作業系統間的溝通橋樑 , 提供基本的操作介面
  • 我們可將 shell 看成是一種使用者環境

  • shell 負責使用者和作業系統兩者之間的

    • 溝通

  • shell也稱為指令解譯器

  • 每一種作業系統都有其特定的 shell
    • MS DOS 的標準 shell 是 command.com
    • Windows 的 shell 是 explorer.exe
    • Unix 之類的作業系統, 其使用的 shell 各有不同

  • 更換 shell 之前, 先認識提示符號並瞭解目前正在使用何種shell
  • 指令提示符號
    • 一般是在使用者目前的目錄加上 $、%、# 或 > 符號
    • 用處是告訴使用者現在可以下達指令

  • 檢視預設的 shell 種類
    • 最簡單的方式是執行 echo 指令, 詢問系統 SHELL 環境變數的值
  • 利用可以查詢使用者資料的 finger 工具, 也能看出該使用者預設的 shell

  • 執行 sudo apt-get install finger 指令安裝工具，再執行 finger 使用者帳號 指令就能查詢使用者資料

  • 更換 shell :最簡單的方法就是直接執行該 shell 之名稱
  • 若要回到原來的 shell, 則執行 exit 指令

  • 接下來讓我們一同看看 bash 的特色

  • 指令歷程都記錄在使用者家目錄裡的 .bash_history 檔案內

  • 執行 history 指令可列出最近使用過的指令及其編號

  • 除了直接指定編號之外, 也能根據 "減法" 原則執行指令

  • 別名的作用可讓使用者自訂新的指令名稱來替代原有的指令

  • 欲得知目前有多少指令被定義了別名, 可執行 alias 指令

  • unalias 指令取消別名

  • 若希望每次登入時系統會自動設定別名,

    • 請將 alias 指令加入該帳號家目錄下的 .bashrc 檔案裡

  • 指令補全
  • 假使輸入資料不足，此時再按一次 Tab 鍵, bash 就會把所有可能的指令都列出來

  • 有效的方法是：將程式或指令放到背景執行

  • &指令

  • bg指令

  • 如果使用者目前有程序在背景執行中, 使用者可以下達 fg指令, 將它從背景中移到前景執行

  • 再登入時, 可用 ps -aux 指令看到程序仍在背景中執行

  • Linux 系統管理很多時候都是在檢視與分析紀錄檔或是處理各種指令的輸出結果
  • 這會用到多個文字搜尋與處理的相關指令

  • 重導 (redirect) 可將某指令的結果輸出到檔案中
  • 它有兩種用法：">“ 和 ">>"

  • 管線 (pipe) 指令的符號是 "|"

  • grep 指令可以搜尋特定字串並顯示出來, 一般用來過濾先前的結果, 避免顯示太多不必要的資訊

  • 若是使用一般權限的使用者執行, 上例的輸出結果會包含許多錯誤訊息
  • 因為沒有指定檔案的副檔名, 所以 s 開頭的目錄也會被搜尋, 您可使用 -s 參數消除

  • 透過管線，將 ls 指令的輸出結果傳給 grep 指令過濾

  • 使用 tee 指令就可以在螢幕上顯示執行結果, 也可以將執行結果存成指定的檔案

  • shell 下有些特殊字元,補系統指令之不足
    • #：標示註解, 這些說明不會被當成指令或程式執行
    • &：背景方式執行,例如執行 sudo updatedb & 指令
    • '：單引號定義引號內為一個完整的字串, 並且讓 shell 不要解讀其中的空白或特殊字元

    • *：對應任何字串、字元或空字串
    • ?：對應任何單一字元
    • .：表示目前所在的目錄
    • ..：表示上一層的目錄
    • ;：可用它將多個指令分隔開來, 讓指令依序執行

    • >：把執行結果輸入一個檔案中
    • >>：把執行結果附加到一個檔案後面
    • \：脫逸符號 (escape), 解除特殊字元的含意

    • |：建立管線, 將前一個指令的執行結果輸入給下一個指令使用
    • ~：使用者登入後所在的目錄, 也就是使用者家目錄
    • [英文字母或數字]：對應括弧中任意範圍的字元

    • 也可以指定字母的範圍
    • 也可以指定數字的範圍

    • [!英文字母或數字]：排除括弧中任意範圍的字元
    • {英文字母或數字}：對應括弧中的任意字元
