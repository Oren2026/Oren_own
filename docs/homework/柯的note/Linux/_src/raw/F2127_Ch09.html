• 許多時候我們需要登入主機工作 , 卻偏偏分身乏術 , 無法坐在主機前面操作
  • 更多時候我們需要在兩台電腦間 傳送檔案資料 , 並且希望可以 “ 安全 " 的傳送 , 不用擔心資料外洩
  • 上述需求,只要架設 SSH 伺服器都可以達成。

  • SSH (Secure SHell) 是指 SSH 通訊協定
  • SSH 伺服器則是透過此協定, 來提供遠端登入及檔案傳輸的服務
  • SSH 協定便在提供遠端登入及檔案傳輸服務時, 增加了將資料加密的機制

  • SSH 協定
    • 常用的是 SSH2, 其使用 22 號通訊埠, 具有以下 3 個優點
    • 可對傳輸中的資料加密
    • 可完全取代 Telnet、FTP 程式及 r 系列指令 (例如 rlogin、rsh、rcp...等)
    • 可轉送其他 X11 和 TCP/IP 的資料

  • 支援 SSH 協定的程式，比較有名的有:
    • SSH Communications Security 公司的 SSH (網址為 http://www.ssh.com/)
    • OpenBSD 組織的 OpenSSH (網址為 http://www.openssh.org/)
  • 各 Linux 發行版所附的SSH 程式也是 OpenSSH

  • 以 sniffit 這個網管程式為例, 示範如何監測
  • 安裝好後請設定, 選擇要監測的網路介面

  • sniffit 會開始監測選定的網路介面

  • FTP 連線在傳輸過程中並沒有加密
  • 透過監測程式可以直接取得使用者傳送的所有資料 (包含使用者的帳號與密碼)
  • 要解決這個問題, 使用有加密的通訊協定是個不錯的選擇

  • 金鑰加密法 (Key Encryption)
  • SSH 用金鑰來加密傳輸的資料, SSH 伺服器上存在著以下 2 種金鑰
    • host key :私有金鑰、公開金鑰
    • server key :預設每小時重新產生一次

  • 成功建立連線後, sshd 會依下列的步驟處理：
    • 若從遠端的終端機視窗登入 SSH 伺服器, 並且成功建立連線後, 會顯示該使用者上一次登入伺服器的時間及 /etc/motd 公告訊息檔, 並記錄本次登入的時間
    • 檢查是否有 /etc/nologin 檔案的存在, 若有則結束連線

  • 成功建立連線後, sshd 會依下列的步驟處理：
    • 切換到該使用者的權限
    • 設定基本的環境變數
    • 讀取使用者家目錄中的 .ssh/environment 檔 (預設沒有此檔案)

  • 成功建立連線後, sshd 會依下列的步驟處理：
    • 切換到使用者家目錄
    • 若使用者家目錄中有 .ssh/rc 指令檔, 則執行此檔案, 否則執行 /etc/ssh/sshrc 指令檔
    • 執行使用者所指定的 shell 或命令
