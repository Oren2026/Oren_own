  • 早期比較多人使用的有 CVS (Concurrent Versioning System) 與 SVN (Subversion)
  • 最近最多人使用的則是 Git
  • 版本控制系統大致可分為兩種類型：主從式管理與分散式管理

  • CVS 與 SVN 就是屬於主從式管理

  • Git 就屬於分散式管理的版本控制系統

  • Git 就屬於分散式管理的版本控制系統

  • Git 是以目錄為基礎來進行版本控制, 該目錄下的檔案、子目錄都會在版本控管的範圍
  • 使用 Git (包含其他版本控制系統) 好處是可以讓您將資料還原到任何一個有紀錄的時間

  • 版本控制的檔案會有 3 種狀態, 像 3 個虛擬的 "區域“
    • 工作目錄 (working directory)：存放原始碼、圖片或文件的目錄
    • 暫存區 (staging area)：您在工作目錄所做的修改要先加入暫存區, 然後才可以加入儲存庫
    • 儲存庫 (repository)：工作目錄裡所做修改的差異最終就是儲存在儲存庫中 (由暫存區加入)

  • 可以建立一個空的目錄來進行版本控制, 也可以在已有內容的目錄下進行版本控制
