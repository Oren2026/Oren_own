⏰ 1｜工作排程簡介

🔄 為什麼需要排程？

電腦有許多程式需要自動啟動或週期性執行，如清理暫存檔、備份系統資料等。本章介紹 Linux 中定期執行工作的方法。

📂 cron 的兩大類工作
  • 系統執行的工作：由 root 管理，記錄在 /etc/crontab
  • 個人執行的工作：由一般使用者設定，透過 crontab 指令管理

⏰ 2｜cron 排程格式

cron 格式：分 時 日 月 週 命令

欄位範圍
分0–59
時0–23
日1–31
月1–12
週0–6（0 = 星期日）

時間範例：

  • 每小時第 1 分：1 * * * * command
  • 每天 04:20：20 4 * * * command
  • 每年 9/8 00:01：1 0 8 9 * command
  • 每週日 04:09：9 4 * * 0 command
  • 每月 1、15 日 02:20：20 2 1,15 * * command
  • 每天 01:00–04:00 每 25 分：25 1-4 * * * command
  • 每 10 分鐘執行一次：*/10 * * * * command
  • 每 10 天（1、11、21、31日）05:35：35 5 */10 * * command

⏰ 3｜crontab 指令

指令說明
crontab -e編輯使用者的 crontab 排程（建議選 nano 作為編輯器）
crontab -l列出目前所有的排程工作
crontab -r刪除使用者所有的排程工作
sudo crontab -l -u benny系統管理者查看指定使用者的排程
sudo crontab -r -u benny系統管理者刪除指定使用者的排程

使用者的 crontab 檔案儲存在 /var/spool/cron/crontabs/（以帳號名稱為檔名），僅 root 可讀寫。

預設執行結果會以 E-mail 寄送給使用者。可透過輸出導向改為寫入紀錄檔。

⏰ 4｜輸出導向

📝 cron 輸出設定

cron 執行結果預設以 E-mail 寄送。可依需求改為寫入紀錄檔或完全不輸出：

寫法說明
command > /path/log.txt將標準輸出導向檔案（覆寫）
command >> /path/log.txt將標準輸出附加到檔案（累加）
command >& /path/log.txt將標準輸出與錯誤訊息全部導向檔案
command >/dev/null 2>&1不儲存任何執行結果及錯誤訊息

⏰ 5｜at 一次性排程

🎯 at 的用途

at 用於設定一次性的排程工作，適合「只在某個時間點執行一次」的情境，如關機前執行備份。

指令說明
at 時間設定一次性排程（進入互動模式輸入命令）
atq 或 at -l查詢目前排程的 at 工作
atrm ID 或 at -d ID刪除指定的 at 工作
sudo systemctl start atd啟動 atd 服務（at 需此 daemon）

⏰ 6｜開機啟動服務

1
systemctl（現代方式）
systemctl enable 服務名稱 讓服務開機時自動啟動；systemctl disable 服務名稱 取消
2
rc.local（傳統方式）
將要執行的程式寫入 /etc/rc.local，再執行 chmod 755 /etc/rc.local 即可
