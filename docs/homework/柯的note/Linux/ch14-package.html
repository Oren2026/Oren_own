📦 1｜DPKG 套件管理系統

🔧 DPKG 的定位

Ubuntu 的軟體管理核心是 DPKG（Debian Package Management System），最早用於 Debian 發行版，Ubuntu 繼承了這套系統。

✨ DPKG 的四大優點
  • 易於安裝、升級與移除套件
  • 強大的查詢功能
  • 能夠進行套件驗證
  • 支援以原始碼形式發行

能被 apt-get 處理的安裝套件稱為 DEB 套件（副檔名 .deb）。

📦 2｜apt-get / apt-cache 安裝

1
更新套件庫
sudo apt update
2
搜尋套件
apt-cache search 關鍵字（如 apt-cache search hex editor）
3
安裝套件
sudo apt-get install 套件名稱

📦 3｜dpkg 安裝本地 DEB 檔

若已從網站下載 .deb 檔案，可用 dpkg 直接安裝：

sudo dpkg -i 套件檔.deb

查詢 DEB 套件資訊：dpkg -I 套件檔.deb

📦 4｜dpkg 查詢已安裝套件

指令說明
dpkg -l列出所有已安裝的套件
dpkg -L 套件名查詢某套件安裝了哪些檔案到系統中
dpkg --search 檔案路徑查某檔案屬於哪個套件（可簡寫為 dpkg -S）

📦 5｜移除與清除套件

指令說明
sudo apt-get remove 套件名移除套件（保留設定檔）
sudo apt-get purge 套件名移除套件並刪除設定檔
sudo apt-get autoremove自動移除不再需要的相依套件

📦 6｜Tarball 原始碼編譯

📦 Tarball 的兩種格式
  • 原始碼格式：以 C/C++ 等程式語言寫成，需編譯
  • 執行檔格式：二進位檔，下載時需注意 OS 版本與 CPU 架構

建議先安裝編譯環境：sudo apt install build-essential

1
./configure
偵測系統環境，產生編譯所需設定檔
2
make
編譯程式
3
make install
安裝編譯好的程式（需 root 權限）
