🔀 1｜版本控制簡介

📋 為什麼需要版本控制？

將原始碼壓縮成單一壓縮檔很難知道每個版本間的差異。版本控制系統（VCS）能幫助管理原始碼的版本變遷。

🔀 兩種版本控制架構
  • 主從式（CVS / SVN）：所有資料集中在中央伺服器
  • 分散式（Git）：每個開發者都有完整的本地儲存庫，無網路也能運作

🔀 2｜Git 三種狀態

🗂️ 工作目錄（Working Directory）

存放原始碼、圖片或文件的目錄，即專案資料夾。

📝 暫存區（Staging Area）

工作目錄的修改要先加入暫存區，才可以提交到儲存庫。

🗃️ 儲存庫（Repository）

工作目錄的修改差異最終儲存在儲存庫中（由暫存區加入）。

🔀 3｜初始設定與初始化

初次使用 Git 前，須先設定使用者身份：

git config --global user.name "您的名字"

git config --global user.email "您的Email"

初始化版本控制目錄：

git init

Git 會在目錄下建立 .git 子目錄用於存放所有版本記錄。

解決中文檔名亂碼：

git config --global core.quotepath false

🔀 4｜四大核心指令

指令用途
git status查看工作目錄自上次提交後的新增、刪除、修改檔案
git add 檔案將工作目錄的變更加入暫存區
git commit -m "訊息"將暫存區的變更寫入儲存庫
git log查詢所有提交過的修改記錄

🔀 5｜還原與版本退回

還原工作目錄到某個時間點：

git checkout SHA值 或 git checkout 分支名稱

還原單一檔案：

git checkout -- 檔案名稱

⚠️ 退回舊版本後，原本的提交紀錄仍存在 .git 目錄中，Git 不會真正刪除任何歷史。

🔀 6｜分支操作

指令說明
git branch查看目前所有分支
git branch 新分支名建立新分支（不影響當前分支）
git checkout 分支名切換到指定分支
git checkout -b 新分支名建立並立即切換到新分支
git merge 分支名將指定分支合併進來

🔀 7｜GitHub 與推送 push

🐙 GitHub 的用途
  • 多人共同維護同一份資料或原始碼
  • 在多台電腦間同步資料

新增遠端節點：

git remote add origin https://github.com/使用者/儲存庫.git

推送至 GitHub：

git push -u origin master

⚠️ GitHub 現已要求使用個人訪問令牌（Personal Access Token）取代密碼進行驗證。

🔀 8｜複製 clone 與同步 pull

從 GitHub 複製儲存庫到本機：

git clone https://github.com/使用者/儲存庫.git

若權限為 Public，不需帳號密碼即可複製；若要提交修改則需設定權限。

從遠端同步更新到本機：

git pull origin master

多人協作時，其他人推送了新內容後，使用 pull 將遠端更新同步到本機。
