🐍 1｜虛擬環境簡介

🔧 為什麼需要虛擬環境？

Python 虛擬環境可以建立多個獨立的 Python 環境，每個環境中可安裝不同的 Python 版本與不同的套件，彼此互不影響。

常見情境：不同專案依賴不同版本的同一套件（如 A 專案需要 TensorFlow 1.x，B 專案需要 2.x），虛擬環境可以完美隔離。

🐍 2｜建立虛擬環境

1
安裝 venv 套件
sudo apt-get install python3-venv
2
建立虛擬環境目錄
python3 -m venv 目錄名稱（如 python3 -m venv python-venv-ffn）
3
檢視目錄架構
可用 tree 目錄名 查看（需先 sudo apt-get install tree）

🐍 3｜啟動與管理環境

啟動虛擬環境：

source 目錄名/bin/activate

啟動後提示符號會變成 (目錄名) 帳號@主機名:~$，表示目前在該虛擬環境中。

離開虛擬環境：

deactivate

🐍 4｜pip 套件管理

指令說明
pip install 套件名安裝套件（如 pip install ffn）
pip list列出目前環境中所有已安裝的套件
pip freeze輸出所有套件名稱及版本（供 requirements.txt 使用）
pip install -r requirements.txt根據 requirements.txt 安裝所有套件

🐍 5｜匯出與複製環境

1
匯出環境套件
pip freeze > requirements.txt
2
在其他機器建立環境
python3 -m venv 新目錄
3
還原套件環境
source 新目錄/bin/activate && pip install -r requirements.txt

💡 這樣可確保不同機器上的 Python 環境完全一致，執行結果也相同。
