🐳 1｜Docker 簡介

📦 Docker 的定位

Docker 是用來準備應用程式執行所需環境（映像檔 image）的工具，其餘硬體需求由原本的作業系統分配。

🐳 Docker vs 虛擬機器
  • 虛擬機器（VM）：自帶完整 OS，資源佔用大、啟動慢
  • Docker：共享主機 OS，效能更好、啟動更快

🐳 2｜安裝 Docker

使用官方腳本自動安裝：

1. 先安裝 curl：sudo apt-get install curl

2. 下載並執行安裝腳本：curl -fsSL https://get.docker.com | sudo sh

將使用者加入 docker 群組（免 root 執行）：

sudo usermod -aG docker 您的帳號

加入後需重新登入，執行 Docker 時才擁有相關權限。

🐳 3｜Docker Hub 映像檔倉庫

📚 Docker Hub（https://hub.docker.com/）

Docker Hub 是 Docker 官方維護的映像檔倉庫，收錄了數萬種常用軟體的映像檔（如 httpd、nginx、mysql、python 等），可線上搜尋並下載使用。

搜尋到喜歡的映像檔後，可用 docker pull 映像檔名 下載到本機。

🐳 4｜下載與啟動容器

1
下載映像檔
docker pull httpd:2.4.54
2
建立網頁目錄
在家目錄建立 public_html/ 並放入 index.html
3
啟動容器
docker run -d -p 8080:80 -v ~/public_html:/usr/local/apache2/htdocs httpd:2.4.54

🐳 5｜容器管理

指令說明
docker ps查看執行中的容器
docker ps -a查看所有容器（含已停止）
docker stop 容器ID正常停止容器
docker start 容器ID啟動已停止的容器
docker rm 容器ID刪除容器（需先停止）
docker rm -f 容器ID強制刪除執行中的容器

🐳 6｜映像檔管理

指令說明
docker images查看本機所有已下載的映像檔
docker rmi 映像檔ID刪除指定映像檔
docker pull 映像檔名下載映像檔
docker inspect 映像檔名查看映像檔詳細資訊

🐳 7｜修改與自訂映像檔

1
以互動模式進入容器
docker run -it httpd:2.4.54 /bin/bash
2
安裝工具並修改設定
如修改 /usr/local/apache2/htdocs/index.html
3
另存為新映像檔
docker commit -m "說明" 容器ID 新映像檔名
