💬 1｜Shell 簡介

🖥️ Shell 的定位

Shell 的功能在於搭起使用者與作業系統間的溝通橋樑，提供基本的操作介面。我們可將 shell 看成是一種使用者環境。

Shell 負責使用者和作業系統兩者之間的溝通，也稱為指令解譯器。

💬 2｜各作業系統的 Shell

每一種作業系統都有其特定的 Shell，了解各系統的預設 Shell 有助於跨平台操作。

作業系統預設 Shell備註
MS DOScommand.com命令列直譯器
Windowsexplorer.exe圖形化殼層
Unix / Linuxbash（多數發行版預設）指令解譯器，功能最強

💬 3｜提示符號與檢視 Shell

⌨️ 提示符號的功能

一般是在使用者目前的目錄加上 $、%、# 或 > 符號，用處是告訴使用者現在可以下達指令。

更換 Shell 之前，先認識提示符號並瞭解目前正在使用何種 Shell。

  • 最簡單的方式：echo $SHELL 詢問系統 SHELL 環境變數的值
  • 使用 finger 使用者帳號 指令查詢（需先 sudo apt-get install finger）

💬 4｜更換預設 Shell

  • 最簡單的方法：直接執行該 Shell 之名稱即可切換
  • 回到原來的 Shell：exit 指令
🔐 限制使用者登入 Shell（chsh）

使用 chsh 指令可更換使用者預設 Shell。將 Shell 換成 /sbin/nologin，即可限定該使用者（如 lambert）不能 telnet 或 SSH 登入。

也可直接修改設定檔來變更預設值。

💬 5｜Bash Shell 簡介

每一種 Shell 都會有些許的不同。Ubuntu 的預設值是 bash（Bourne Again Shell），以下讓我們一同看看 bash 的特色。

💬 6｜指令歷程 (History)

  • 指令歷程記錄在家目錄的 .bash_history檔案內
  • history 指令可列出最近使用過的指令及其編號
  • 除了直接指定編號之外，也能根據「減法」原則執行指令（如 !-2 執行倒數第二個指令）

💬 7｜別名 (Alias)

🔖 什麼是別名？

別名的作用可讓使用者自訂新的指令名稱來替代原有的指令。

  • 查看目前所有別名：alias
  • 取消別名：unalias 別名稱
  • 每次登入自動設定：將 alias 指令加入家目錄的 .bashrc 檔案裡

💬 8｜指令補全 (Tab)

按一次 Tab 鍵可補全指令或檔案名稱。若輸入資料不足，再按一次 Tab，bash 就會把所有可能的指令或檔案都列出來。

💬 9｜背景執行 (fg / bg)

有些時候我們所使用的程式或指令需要長時間執行，有效的方法是：將程式或指令放到背景執行。

指令說明
指令 &在指令最後加 &，直接丢到背景執行
bg將暂停的程序搬到背景繼續執行
fg將背景程序移到前景執行
ps -aux檢視目前所有在背景執行的程序

💬 10｜系統分析指令

Linux 系統管理很多時候都是在檢視與分析紀錄檔或是處理各種指令的輸出結果，這會用到多個文字搜尋與處理的相關指令。

💬 11｜重導 (Redirect) 與 管線 (Pipe)

符號名稱功能範例
>輸出重導將執行結果寫入檔案（覆寫）ls > result.txt
>>附加重導將執行結果附加到檔案後ls >> result.txt
|管線將前一個指令的輸出傳給下一個指令ls | grep log
  • grep：搜尋特定字串並顯示，可過濾結果
  • -s 參數：消除無權限存取的錯誤訊息
  • tee：同時在螢幕顯示並存入檔案

💬 12｜Shell 特殊字元

分類字元說明
註解#標示註解，不會被當成指令執行
執行&背景方式執行，如 sudo updatedb &
引號'...'單引號內為完整字串，Shell 不解讀空白或特殊字元
"..."雙引號，允許變數替換（$var）
萬用字元*對應任何字串、字元或空字串
?對應任何單一字元
路徑.目前所在的目錄
..上一層目錄
範圍[abc]對應括弧中任意單一字元
排除[!abc]排除括弧中任意字元
I/O>、>>輸出導向（覆寫／附加）
|建立管線
脫逸\解除特殊字元的含意
其他;、~; 分隔多指令依序執行；~ 為家目錄簡稱

💬 13｜環境變數

🔧 環境變數 (Environment Variables)

Shell 本身有一組用來儲存系統資訊的變數，稱為環境變數。環境變數依據 Shell 種類的不同，會有不一樣的變數及設定方法。

1
檢視
set（Bash） / setenv（tcsh）
2
設定（Bourne Shell 家族）
export VAR_NAME=value
3
永久生效
編輯家目錄的 .bashrc 檔案

💬 14｜PATH 環境變數

有時候執行某些程式時，會出現指令找不到的情形。這是因為每個使用者的家目錄以及其他目錄預設並沒有放入 PATH 環境變數中。

📁 執行現行目錄下的程式

兩種方式：

  • 明確指定路徑：./程式名稱
  • 將現行目錄加入 PATH：export PATH=$PATH:.
