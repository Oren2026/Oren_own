💾 1｜掛載 USB 裝置

🔌 掛載概念

掛載（mount）就是將儲存媒體（光碟、USB、硬碟）指定成系統中的某個目錄。此目錄稱為掛載點。

1
查詢 USB 裝置名稱
先用 fdisk -l 或 lsblk 查詢 USB 的裝置代號（如 /dev/sdb1）
2
建立掛載點
sudo mkdir /media/usbdisk（預設無此目錄，需自行建立）
3
掛載
sudo mount /dev/sdb1 /media/usbdisk
4
卸載
sudo umount /media/usbdisk（卸載後才能拔出裝置）

一般使用者掛載權限：Linux 預設僅 root 可執行 mount。可編輯 /etc/sudoers 開放一般使用者。

若無法卸載：fuser -u /media/usbdisk/ 可查出使用該目錄的程序與使用者。

💾 2｜掛載光碟與 ISO 檔

  • 掛載光碟機後，執行 umount 卸載後才能取出光碟片
  • 直接存取 ISO 檔內容：使用 mount -o loop image.iso /mnt/iso 掛載 ISO 檔為目錄，無需解壓縮即可讀取內部檔案

💾 3｜檢查磁碟使用狀況

指令用途範例
df -h查詢磁碟剩餘空間（數值人性化）Human-readable 顯示
df -T顯示檔案系統格式ext3、ext4、ntfs 等
dstat監控磁碟讀寫狀況dstat -c CPU、-n 網路
iotop查看哪個使用者/程式在使用磁碟 I/Oiotop -u username
du -sh查詢目錄所佔硬碟空間du -sh /var

💾 4｜連結與符號連結

🔗 硬連結 vs 符號連結

在 Linux 中，有兩個（或以上）檔案使用相同 inode，稱為硬連結 (hard link)。而符號連結 (symbolic link / symlink) 則是儲存另一個檔案的路徑。

特性硬連結 (ln)符號連結 (ln -s)
inode兩檔案使用相同 inode各自擁有獨立的 inode
原檔刪除後仍可繼續使用失效（像斷掉的捷徑）
跨分割區不可（必須同一分割區）可（可指向任何位置）
指向目錄不可可
佔用空間幾乎不佔額外空間佔用一個 inode
建立指令ln 原始檔 連結名ln -s 原始檔 連結名

查詢 inode 編號：ls -i 指令可觀察檔案的 inode 編號。

版本相關檔案（如 /etc/nginx/sites-enabled 指向 sites-available）通常使用符號連結。

Windows 上可使用 mklink 指令建立連結。

💾 5｜安裝第 2 顆硬碟

1
硬碟分割
使用 fdisk /dev/sdb 指令分割新硬碟
2
格式化
mkfs.ext4 /dev/sdb1 格式化為 ext4
3
掛載
mount /dev/sdb1 /mnt/newdisk
4
開機自動掛載
修改 /etc/fstab 檔，加入開機時自動掛載

💾 6｜Swap 分割區

💠 Swap 的角色

Swap（置換空間）是當實體記憶體不足時，系統用磁碟空間代替記憶體使用的機制。Ubuntu 安裝時若選擇自動分割，預設會在根目錄建立一個 swapfile（而非獨立分割區）。

操作指令
新增 Swap 分割區fdisk 新增分割區，設定類型為 swap
格式化 Swapmkswap /dev/sdaX
啟動 Swapswapon /dev/sdaX
檢查 Swapswapon -s 或 free
關閉 Swapswapoff /dev/sdaX
建立 Swap 檔案dd if=/dev/zero of=myswap bs=1M count=1024
開機自動啟動修改 /etc/fstab 加入 Swap 項目
