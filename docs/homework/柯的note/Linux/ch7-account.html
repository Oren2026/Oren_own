👤 1｜認識使用者帳號

🖥️ 使用者帳號的必要性

Linux 屬於多人多工的作業系統，可讓不同使用者從本地端或遠端登入。無論本機或遠端登入，都必須在該主機上擁有帳號。

🛡️ 帳號的三大功能
  • 掌控使用者身份——區分不同使用者
  • 資源共享，不互相干擾——家目錄、檔案權限隔離
  • 權限管制，維護系統安全——防止未授權操作

👤 2｜/etc/passwd 檔案

每個使用者帳號在 /etc/passwd 檔案中都有 7 個欄位。

欄位說明
1. 帳號名稱使用者登入時的名稱
2. 使用者密碼預設為 x（密碼移至 shadow 檔案）
3. UID（User ID）使用者識別碼，root = 0
4. GID（Group ID）主要群組識別碼
5. 使用者相關資訊如姓名、电话等（可空白）
6. 使用者家目錄如 /home/lambert
7. 使用者環境預設 Shell，如 /bin/bash

預設情況下，任何使用者都可以讀取 /etc/passwd（無法竊取密碼，因為密碼已移至 /etc/shadow）。若將權限改為 rw-------，則僅 root 可讀取。

👤 3｜/etc/shadow 檔案

🔑 shadow 檔案的目的

將密碼雜湊值從 /etc/passwd 分離至 /etc/shadow，僅 root 可讀取，大幅提升安全性。

欄位說明
1. 帳號名稱與 passwd 對應
2. 密碼雜湊後的密碼（$6$... 格式）
3. 密碼最後變動時間距離 1970-01-01 的天數
4. 密碼最小間隔密碼變動後，需隔多久才能再改（天數）
5. 密碼最大有效期密碼失效前必須更換的天數
6. 警告期密碼即將過期前多少天開始警告
7. 停權期限密碼過期後，帳號被停用的天數
8. 帳號使用期限帳號本身的有效日期
9. 保留欄位預設空白

👤 4｜建立帳號 useradd

1
useradd 新增
執行 useradd 帳號名稱，系統自動更新 /etc/passwd 與 /etc/shadow
2
passwd 設定密碼
執行 passwd 帳號名稱 為新帳號設定密碼
3
預設值設定檔
useradd 的預設值記錄在 /etc/login.defs，家目錄位置、環境、樣板目錄則在 /etc/default/useradd

帳號名稱必須唯一。若名稱已被佔用，系統會回報「使用者已存在」。可用以下三種方式事先查詢：

  • 建立資料庫自行查詢
  • 直接查 /etc/passwd 檔案
  • 執行 finger 帳號（需安裝）

👤 5｜管理者 root 與 sudo

👑 root 帳號

Linux 安裝後即建立 root 帳號。root 為系統管理者，對系統擁有完全控制權，可做任何設定和修改。維護 root 帳號的安全便格外重要。

🔐 sudo 機制

一般使用者可透過 sudo 指令 以 root 權限執行特定指令。若新建帳號無法使用 sudo，需將其加入 /etc/group 中的 sudo 或 wheel 群組。

緊急修復：重設 root 密碼（GRUB 開機選單 → 按 Ctrl+x 或 F10 進入單人模式 → passwd root）

鎖定系統維護：在 /etc 目錄執行 touch nologin，可阻止一般使用者登入。

👤 6｜帳號切換 su

su（switch user）指令可切換身份。例如 su - lambert 可切換至 lambert 帳號，並同時切換至該帳號的家目錄和環境變數。

  • su 帳號：僅切換身份，保留目前目錄
  • su - 帳號：連同工作目錄和環境變數一起切換
  • exit：切換回原帳號

👤 7｜停用與刪除帳號

1
停用帳號（保留資料）
編輯 /etc/passwd，將該帳號的 Shell 改為 /sbin/nologin 或在 shadow 中設定密碼過期
2
刪除前準備
先刪除該使用者的程式（ps -u uid + kill PID）和排程工作（sudo crontab -u 使用者名稱 -r）
3
刪除帳號
執行 userdel -r 帳號名稱（-r 同時刪除家目錄和郵件）

常見情境：員工畢業、離職或逾期不繳費時，停用或刪除該帳號以阻止繼續登入系統。

👤 8｜群組管理

👥 群組的目的

可依不同性質自行建立群組。當檔案或目錄隸屬於某群組時，該群組的使用者會受到群組權限的限制。一個使用者可同時隸屬於多個群組。

/etc/group 欄位說明
群組名稱如 sudo、www-data
群組密碼通常為空白或 x
GID群組識別碼
成員列表隸屬此群組的帳號（以逗號分隔）

建立群組：groupadd 群組名稱
