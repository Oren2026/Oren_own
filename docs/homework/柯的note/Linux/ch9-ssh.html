🔐 1｜SSH 簡介

🌍 SSH 的用途

SSH（Secure SHell）可達成兩件事：遠端登入主機（解決分身乏術的問題）與安全傳送檔案（加密傳輸，不怕資料外洩）。

✨ SSH2 的三大優點
  • 可對傳輸中的資料加密
  • 可完全取代 Telnet、FTP、r 系列指令（rlogin、rsh、rcp）
  • 可轉送其他 X11 和 TCP/IP 的資料

SSH2 預設使用 22 號通訊埠。Linux 發行版附的 SSH 程式皆為 OpenSSH（http://www.openssh.org/）。

🔐 2｜SSH vs FTP 安全性

⚠️ FTP 的安全問題

FTP 連線在傳輸過程中沒有加密，透過監測程式可直接取得使用者傳送的帳號與密碼。

解決方案：使用有加密的通訊協定，例如 SSH（scp / sftp）或 HTTPS。

🔐 3｜金鑰加密原理

🔑 SSH 伺服器的兩種金鑰

SSH 伺服器上存在兩種金鑰，用於加密傳輸資料：

金鑰類型說明
Host Key（主機金鑰）包含私有金鑰與公開金鑰，伺服器身份驗證用
Server Key（伺服器金鑰）預設每小時重新產生一次，提高安全性

🔐 4｜sshd 連線流程

用戶端連線成功後，sshd 依序執行以下步驟：

1
登入紀錄
顯示上次登入時間及 /etc/motd 公告訊息檔，並記錄本次登入時間
2
檢查停權
檢查是否有 /etc/nologin 檔案，若有則立即結束連線
3
切換權限
切換至該使用者的權限，設定基本環境變數，讀取 ~/.ssh/environment（若存在）
4
執行 shell
切換至家目錄，若有 ~/.ssh/rc 則執行之，否則執行 /etc/ssh/sshrc，最後啟動使用者的 shell 或指定命令

🔐 5｜安裝與基本使用

安裝 openssh-server：sudo apt install openssh-server openssh-client

指令用途
ssh 帳號@主機遠端登入（slogin 為符號連結）
scp遠端複製檔案（secure copy）
sftp遠端傳輸檔案，互動模式上傳/下載
ssh-keygen產生與管理 RSA/ECDSA 金鑰
ssh-agentSSH 認證代理程式，記憶通行密碼
ssh-add將私有金鑰加入代理程式
ssh-keyscan收集別台主機的 SSH 公開金鑰
sshdSSH 伺服端程式（ daemon）

🔐 6｜scp 遠端複製

下載檔案：

scp 帳號@伺服器:檔案路徑 本地路徑

範例：scp tony@server:/home/tony/report.txt /tmp/

上傳檔案：

scp 本地檔案 帳號@伺服器:目標路徑

範例：scp myfile.txt benny@server:/home/benny/

可一次指定多台伺服器，一次完成多點傳輸。

🔐 7｜sftp 互動模式

📂 sftp 基本操作

sftp 採用互動交談方式，使用 put 和 get 指令上傳與下載檔案，連線全程加密。

Windows 內建 SFTP：Windows 10/11 可在 CMD 中直接使用 sftp 指令。

🔐 8｜密碼認證

基本語法：

ssh 帳號@主機（互動式輸入密碼）

ssh 帳號@主機 指令（直接執行遠端指令後斷線）

若出現錯誤訊息，檢查伺服器是否已啟動 sshd 且通訊埠 22 未被阻擋。

🔐 9｜金鑰認證

1
產生金鑰
ssh-keygen -t rsa（RSA 加密）或 ssh-keygen -t ecdsa（ECDSA）
2
本機測試
本機先測試：ssh-copy-id localhost 將公鑰寫入 ~/.ssh/authorized_keys
3
複製到遠端
將本機 ~/.ssh/id_rsa.pub 內容複製到遠端 ~/.ssh/authorized_keys

無密碼登入：若不想每次輸入密碼，可產生不含通行密碼的金鑰。建立連線後直接登入，不要求任何密碼。

代理程式：ssh-agent 可幫我們記憶通行密碼，ssh-add 將私有金鑰加入代理。
