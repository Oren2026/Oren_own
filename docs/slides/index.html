🚧 挑戰一：CRON 卡住

❌ 問題

  • 每小時整合停在 "running"
  • GitHub 24 小時沒更新
  • daily/ 累積 5+ 篇文章
  • 系統看似正常但實際僵屍

✅ 解法

  • 刪除僵住的 CRON
  • 重新建立新的每小時任務
  • 設定 120 秒 timeout
  • 未來加強監控機制
3 / 8

🚧 挑戰二：Telegram 回應延遲

❌ 問題

  • 14:00 後需要發 2 次才能回
  • 溝通節奏被打斷
  • 讓人覺得「系統怪怪的」

✅ 解法

  • 確認 isolated session 隔離
  • CRON 與 main session 獨立
  • 觀察是否網路/頻道問題
  • 目前已穩定，持續監控
4 / 8

🚧 挑戰三：標籤太雜

❌ 問題

  • 所有標籤都顯示出來
  • 數量多但很多只出現 1-2 次
  • 過濾器反而造成視覺負擔

✅ 解法

  • 設定門檻：只顯示 ≥3 篇的標籤
  • 動態計算，隨文章增加調整
  • 上方乾淨，點下面文章才看到

🎨 排版與設計的邏輯

🧱 結構優先

  • 先有層次，才有美觀
  • 資訊太多時用過濾/分類
  • 手機 vs 電腦不同佈局

👁️ 視覺引導

  • 顏色用來區分狀態
  • 大小用來表示重要性
  • 留白讓眼睛休息

🔄 互動回饋

  • 點擊要有反應
  • 狀態變化要明確
  • 操作要可逆/可恢復

📱 簡約取勝

  • 不重要的就隱藏
  • 門檻式呈現（≥3才顯示）
  • 減少視覺噪音
