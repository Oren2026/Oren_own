# v0.1 — 現狀分析

> 日期：2026-05-08
> 狀態：現狀記錄

---

## 現有架構問題

### 1. main.py 是 Monolithic

```python
# main.py 第 16-57 行
# 所有模組在啟動時全部初始化
ball_detector = BallDetector()
arm = HIWINArm()
arduino = ArduinoController()
strike_ctrl = StrikeController(arm, arduino)
# ...
```

**問題**：
- 任何一個模組 import 失敗 → 整支程式炸掉
- 新增模組 = 修改 main.py
- 無法單獨測試某個模組

### 2. Star Import 污染 Namespace

```python
from config import *
from vision import BallDetector, CueAligner, TableTransform
from comm import HIWINArm, ArduinoController
```

**問題**：
- 變數名稱衝突難追
- IDE 無法追蹤誰定義了什麼

### 3. UI 不是真正的 UI

- `Dashboard` 只是 `cv2.imshow` 包裝，blocking
- `TunePanel` 是 blocking CLI loop，無法與 webcam loop 同時運行
- 沒有脫離 main while loop 的控制權

### 4. 硬耦合

```python
# main.py 第 43 行
strike_ctrl = StrikeController(arm, arduino)
# motion 直接依賴 comm 的實例
```

**問題**：
- 改 comm 介面 → motion 跟著要改
- 測試 motion → 必須要有 comm 和手臂實際連線

### 5. 跨模組資料用 dict

```python
# main.py 第 76-83 行
status = {
    "ball_found": ball_info["found"],
    "ball_x": ball_info["ball_x"],
    # ...
}
```

**問題**：
- 沒有型別檢查
- 拼錯 key 名稱不會報錯
- 文件化靠閱讀 code
