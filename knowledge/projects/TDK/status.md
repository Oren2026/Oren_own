## 基本資訊
- **比賽時間**: 2026-11（初審/複審）
- **參賽組別**: 遙控組
- **比賽主題**: 讓世界看見雲林
- **目前階段**: 機構設計階段

## 比賽任務（四大關卡）

### 關卡1｜成龍濕地生態守護
- 扶正裝置藝術（籃子、杯子等）並放置於定點
- 救護候鳥娃娃（尖叫雞、天鵝）送往救護站
- 機構：叉子從側面/下方托持或夾住短邊（40cm側）

### 關卡2｜文蛤分級
- 將文蛤掃入箱子 → 晃動篩選 → 倒入下一箱 → 再次晃動 → 倒入最終盤
- 機構：叉子叉住箱子邊緣操作

### 關卡3｜稻草卷堆放
- 將稻草卷堆成三角形結構
- 機構：叉子串入草捆（穿刺方式提起）

### 關卡4｜北港迎媽祖
- 水果（文旦/柳丁/砂糖橘）放入供桌盤
- 擲筊動作
- 機構：叉子夾持水果（8/12/18cm 三尺寸）

## 爪子機構設計

### 核心思路
- 爪子本體固定，不更換零件
- 同一支叉子，根據任務用不同介入策略：
  - 穿刺（稻草卷）
  - 托住（娃娃、箱子）
  - 夾持（水果、裝置藝術）
- 操作者視覺判斷，機構靠人為補足細節

### 爪子結構
- 形式：叉子型（多叉齒）
- 單支爪子，多接觸點，選用不同齒位達成不同間距效果
- 原則間距：~12cm（稻草卷），實際應變再微調

## 軟體模組規劃

### 核心重複利用模組
| 模組 | 用途 |
|------|------|
| 力道控制模組 | 閉環力道偵測，適用於所有夾持/穿刺任務 |
| Z軸升降控制 | 高度對位，關卡1/3/4都需要 |
| 任務序列引擎 | 預定義動作序列，驅動關卡2/3/4多階段任務 |
| 移動協同API | 底盤移動與爪子動作同步控制 |
| 傾倒角度控制 | 關卡2箱子操作、關卡4水果放置 |

### 各關卡專屬功能
- 關卡1A：垂直扶正施力策略
- 關卡2：晃動頻率控制（模擬人工篩選節奏）
- 關卡3：三角形堆疊定位算法
- 關卡4B：擲筊動作模擬

## 軟體分層架構
```
應用層（各關卡任務邏輯）
    ↓
任務序列引擎（預定義動作排程）
    ↓
爪子控制API（grip/release/tilt/lift/swing/posture）
    ↓
硬體抽象層（Mega2560 PWM / 力道感測 / 超聲波讀取）
```
