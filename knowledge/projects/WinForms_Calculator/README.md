# 📜 WinForms 現代化計算機開發實錄

這是一個基於 C# WinForms 開發的計算機練習專案。不同於傳統計算機，本專案專注於非線性輸入體驗與高容錯邏輯設計，透過與 AI 的深度協作，將複雜的運算式解析轉化為直覺的操作行為。

---

## 🚀 核心設計思維 (Core Logic)

本專案拒絕傳統計算機「按一步算一步」的限制，改採「全運算式預覽」模式：

- **雙層顯示架構**：上方 TextBox 承載完整運算式（如 `2+(3×4)-5`），下方實時回傳計算結果。
- **非線性修正**：支援 Backspace 逐字刪除與運算子自動覆蓋邏輯，確保輸入流暢不中斷。

---

## 🛠️ 技術亮點與容錯機制

為了提升生產效能，我們在後端邏輯中實作了以下自動化處理：

### 1. 智慧運算式修正

| 機制 | 說明 |
|------|------|
| **自動補齊括號** | 使用者無需擔心未閉合的括號（如輸入 `(5+3` 直接按 `=`），系統會自動閉合並執行運算。 |
| **運算子覆蓋 (Operator Overriding)** | 防止 `2++3` 這類無效輸入。當連續按下運算子時，系統會自動保留最後一個動作。 |
| **末尾自動過濾** | 按下等號時，若運算式末尾殘留運算子（如 `5+3+`），系統會自動忽略末尾符號以確保計算成功。 |

### 2. UI/UX 響應式佈局

- **TableLayoutPanel 容器**：利用表格佈局確保視窗在縮放時，按鈕與顯示區能保持完美的比例與間距。
- **狀態分流設計**：
  - 主顯示區：永遠保留有效的數值，不被 "Error" 字眼佔據。
  - 狀態列 (Status Bar)：將技術性的錯誤提示（如語法錯誤）移至底部狀態列，減少對使用者視覺的干擾。

## 💡 互動特點

| 功能 | 描述 |
|------|------|
| **即時運算 (Real-time)** | 每一次按鍵點擊都會觸發解析器，除非運算式不完整，否則結果即時呈現。 |
| **混合輸入** | 完整支援實體鍵盤對應與滑鼠點擊，提升輸入效率。 |
| **靜默出錯** | 運算式出錯時，介面會保留上一次正確的結果，而非直接崩潰或清空。 |

## 🧠 設計演進紀錄

這款專案是開發者與 AI 針對「一般行為」與「進階需求」反覆辯證後的產物：

1. **需求定義**：從簡單四則運算演進至支援複雜巢狀括號。
2. **邏輯迭代**：解決了 WinForms 在縮放時容易跑版的問題，並優化了運算式解析的穩定度。
3. **優化體驗**：將錯誤處理從「阻斷式」改為「提示式」，讓操作手感更貼近專業科學計算機。
